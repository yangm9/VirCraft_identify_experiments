# 08_run_vircraft

VirCraft Viral Identification Experiment - Train/Val/Test Pipeline

## Directory Structure

├── mk_identify_data.sh # Main pipeline script
├── train_sample.list # Training set sample list
├── val_sample.list # Validation set sample list
├── test_sample.list # Test set sample list
├── train/ # Training set output directory (generated after execution)
├── val/ # Validation set output directory (generated after execution)
└── test/ # Test set output directory (generated after execution)


## Dataset Split

The experiment contains three independent datasets, each with 8 samples:

| Dataset | Sample List File | Positive Samples | Negative Samples | Fragment Length Categories |
|---------|-----------------|------------------|------------------|---------------------------|
| Training | `train_sample.list` | positive_* | negative_* | 1-2k, 2-5k, 5-10k, 10-20k |
| Validation | `val_sample.list` | positive_* | negative_* | 1-2k, 2-5k, 5-10k, 10-20k |
| Test | `test_sample.list` | positive_* | negative_* | 1-2k, 2-5k, 5-10k, 10-20k |

Sample naming convention:
- `positive_*`: Positive samples (contain viral sequences)
- `negative_*`: Negative samples (no viral sequences)
- Suffix indicates fragment length range, e.g., `1-2k` means 1kb-2kb sequence fragments

## Script Description

### `mk_identify_data.sh`

This script performs the following three steps sequentially for each sample in each dataset:

1. **VirCraft Identification**: Uses `virCraft.py identify` to identify viral sequences
2. **CheckV Quality Control**: Uses `checkv end_to_end` for quality assessment of identified results
3. **Proviral Prediction**: Uses `proviral_predict.py` to predict proviral integration sites

#### Key Parameters

| Parameter | Value | Description |
|-----------|-------|-------------|
| `-f` | permissive | Permissive filtering mode |
| `-m` | vs2-vb-dvf-gn | Database combination used |
| `-l` | 1000 | Minimum sequence length (bp) |
| `-t` | 64 | Number of threads |
| `-d` | ~/database/checkvDB/checkv-db-v1.5 | CheckV database path |

#### Input File Path Convention

The script assumes input fasta files are located at:

`../07_fragment_subsample/{train|val|test}/{sample_name}.fasta`


#### Output Directory Structure

Each sample's output is located at:

├── work_files/
│ ├── candidate_viral_ctgs.info.tsv # VirCraft candidate viral results
│ └── checkv_all/ # CheckV output directory
└── ... (other VirCraft output files)

## Important Notes

1. Runtime: Each sample requires three analysis steps using 64 threads. Runtime may be substantial depending on the number of sequences.
2. Disk Space: CheckV and VirCraft generate many intermediate files. Ensure sufficient disk space is available.
3. Path Dependencies: The script uses relative path ../07_fragment_subsample/. Ensure your current directory structure matches:
07_fragment_subsample/   (parent directory)
08_run_vircraft/         (current directory)
4. Conda Initialization: The script reloads conda in each loop iteration to ensure proper environment activation.
